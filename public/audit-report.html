1. Structure & Architecture 8/10

9 composants bien découpés, build statique SSG, TypeScript strict

Points forts
  • Architecture propre : 1 page, 9 composants spécialisés
  • "use client" utilisé uniquement sur les 3 composants interactifs
  • Alias @/* configuré, imports propres
  • Build statique (SSG) — pages pré-rendues
À améliorer
  • Footer contient le formulaire Contact — devrait être un composant séparé
  • Pas de robots.txt ni sitemap.xml
  • Pas de favicon.ico dans /public
  • Pas de page 404 personnalisée (not-found.tsx)
  • ESLint non configuré (pas de config à la racine)

2. SEO & Metadata 5/10

Title et description OK, mais pas d'Open Graph, pas de sitemap, pas de structured data

Points forts
  • lang="fr" correctement défini
  • Title et meta description pertinents
  • Keywords ISAK / anthropométrie / sport
  • Un seul <h1>, hiérarchie correcte
À améliorer
  • Pas d'Open Graph (og:title, og:description, og:image)
  • Pas de Twitter Card
  • Pas de canonical URL
  • Pas de structured data JSON-LD (Schema.org)
  • Pas de robots.txt ni sitemap.xml
  • Pas de favicon via metadata Next.js
  • Google Fonts via <link> au lieu de next/font

3. Performance 6/10

Bundle 113 kB First Load — bon, mais fonts et animations à optimiser

Points forts
  • Build statique (SSG) — temps de réponse minimal
  • Bundle JS : 113 kB First Load (raisonnable)
  • Utilisation de next/image (optimisation auto)
  • Tailwind CSS v4 tree-shaking efficace
À améliorer
  • Google Fonts via <link> bloque le rendu — migrer vers next/font/google
  • Animations CSS continues (pulse-slow, scanline) tournent hors viewport
  • SVG inline abondants dans le JS — pourraient être des fichiers séparés
  • Pas de priority sur les images above-the-fold

4. Accessibilité (a11y) 4/10

Problèmes multiples : labels, modale, aria, contrastes

Points forts
  • aria-label="Menu" sur le bouton hamburger
  • Sémantique HTML correcte (<nav>, <main>, <footer>)
  • Alt text sur les 2 images (About)
À corriger
  • Labels formulaire sans htmlFor/id
  • Contraste text-muted (#8d99ae) sur #151515 : ~4.1:1 (limite AA)
  • Bouton menu sans aria-expanded
  • Modale Dashboard : pas de focus trap, pas de role="dialog", pas de fermeture Escape
  • SVG décoratives sans aria-hidden="true"
  • Pas de skip link
  • Lien logo href="#" ne mène nulle part

5. Responsive & UI/UX 8/10

Points forts
  • Grid responsive (1/2/3 cols selon breakpoint)
  • Menu hamburger mobile fonctionnel
  • Typography responsive (4xl → 5xl → 7xl)
  • Design system cohérent, accent rouge carmin unifié
  • CTAs bien visibles et contrastés
À améliorer

  • Pas de bouton scroll-to-top (page très longue)
  • Pas d'animations d'entrée au scroll

6. Sécurité 7/10

Surface d'attaque minimale (SSG), mais formulaire non branché

Points forts
  • Site statique — pas de route API, pas de BDD
  • Pas de données sensibles exposées
  • Pas de dépendances tierces côté client
À améliorer
  • Formulaire non connecté (juste setSubmitted(true))
  • Pas de protection anti-spam (honeypot / CAPTCHA)
  • Pas de Content Security Policy dans next.config.ts
  • Email en clair (susceptible au scraping)

  • SVGs inline dans les données (mélange data/JSX)
